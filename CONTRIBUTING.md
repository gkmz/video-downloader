# Contributing to VDD

感谢你关注 VDD。欢迎通过 Issue 和 Pull Request 参与改进。

## 贡献范围

你可以从以下方向参与：

- Bug 修复
- 功能改进与体验优化
- 文档修订
- 测试补充与质量提升
- 构建与发布流程优化

## 开始之前

1. 先阅读 [README](./README.md) 与相关文档。
2. 先搜索已有 Issue，避免重复工作。
3. 涉及较大改动时，建议先开 Issue 讨论方案。

## 开发环境

- Go 版本：以 [go.mod](./go.mod) 为准
- 运行依赖：`yt-dlp`、`ffmpeg`

常用命令：

```bash
go mod download
go build ./...
go test ./...
```

说明：部分测试依赖外网资源，在离线或受限网络下可能失败。

## 分支与提交建议

- 从 `main` 拉取最新代码后创建功能分支。
- 建议使用清晰的提交信息，例如：
  - `fix: handle empty parser config`
  - `docs: update user guide for open-source edition`
  - `refactor: simplify config initialization`

## Pull Request 要求

提交原则：

- 一个 PR 只解决一个问题（One PR, One Problem）。
- 不要在同一 PR 混入多个不相关目标（例如：功能修复 + 大规模重构 + 文档改写）。
- 若需求较大，请拆分为多个可独立评审与回滚的 PR。

提交 PR 时请尽量包含：

- 变更目的和背景
- 关键实现说明（涉及行为变化时）
- 测试说明（执行了哪些命令、结果如何）
- 兼容性说明（若影响配置、数据、平台行为）

合并前请确保：

- 能通过 `go build ./...`
- 代码与文档一致
- 没有提交构建产物、日志、数据库文件

## 代码风格

- 遵循 Go 官方风格，提交前运行 `gofmt`。
- 保持改动聚焦，避免在同一 PR 混入无关重构。
- 优先修复根因，避免仅修补表面现象。

## 文档与语言

- 用户文档优先使用简体中文。
- 如补充英文内容，请确保术语一致、表达准确。

## 行为准则

请保持尊重、专业、建设性的协作方式。

## 不建议提交的内容

- 大体积二进制
- 构建输出目录
- 本地日志
- 本地数据库文件
- 与本 PR 目标无关的格式化大改

## 许可证

向本仓库提交代码即表示你同意该贡献在本项目许可证（Apache-2.0）下发布。
