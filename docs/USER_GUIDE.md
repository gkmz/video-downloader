# VDD (Video Downloader) 用户使用手册

欢迎使用 **VDD**，这是一款高效、跨平台的视频下载工具，基于 `yt-dlp` 内核，支持 YouTube、Bilibili、TikTok、X (Twitter)、Facebook 等全球主流视频平台。

---

## 🚀 1. 快速开始

### 1.1 安装与启动

- **Windows**: 解压后双击 `vdd-windows-amd64.exe`。
- **macOS**: 双击 `VDD.app` 或 `vdd-darwin-arm64` / `vdd-darwin-amd64`。
  - _注意：首次打开如遇安全提示，请按住 `Control` 键点击图标，选择“打开”。_
- **Linux**: 在终端运行 `./vdd-linux-amd64`。

### 1.2 界面概览

- **地址栏**：粘贴视频链接。
- **解析按钮**：点击开始获取视频信息。
- **下载列表**：显示当前下载进度。
- **状态栏**：底部显示实时状态。
- **工具栏**（右上角）：
  - 🕒 **历史记录**：查看已下载文件。
  - ⚙️ **设置**：配置代理、下载目录等。
  - ℹ️ **关于**：软件信息与版本更新入口。

## 📥 2. 基础功能

### 2.1 下载视频

1. **复制链接**：在浏览器中复制想下载的视频 URL。
2. **自动识别**：如果开启了“监听剪贴板”（默认开启），回到 VDD 时链接会自动填入。
   - 也可以手动粘贴到输入框。
3. **解析**：点击“**解析**”按钮。
4. **选择格式**：解析成功后，会出现一个视频源列表，展示该视频可用的所有画质（如 4K, 1080P, 720P）和音质。
   - 点击某一行的 **“下载”** 按钮即可开始下载。

### 2.2 自动下载

- 在 **设置** 中开启 **“自动下载推荐格式”**。
- 开启后，点击“解析”后将直接下载推荐的画质（带音频）。

### 2.3 查看历史

- 点击右上角 🕒 图标。
- 可以看到所有已下载的视频、大小及路径。
- 双击某一行，或点击“打开”按钮，可直接在文件管理器中定位到该文件。

## ⚙️ 3. 高级设置

### 3.1 文件名格式 (Filename Format)

您可以在设置中自定义下载文件的命名规则：

- **视频标题.ext** (默认)
- **视频标题 - 发布者.ext**
- **发布者 - 视频标题.ext**
- **发布日期 - 视频标题.ext**

### 3.2 代理设置 (Proxy)

如果您需要下载 YouTube 等海外平台视频，请配置网络代理：

- **格式**：
  - HTTP 代理：`http://127.0.0.1:7890`
  - SOCKS5 代理：`socks5://127.0.0.1:1080`
- 请根据您的 VPN/代理软件查看具体端口号。

### 3.3 Cookie 设置 (身份验证)

下载会员专属或有年龄限制的视频（如 Bilibili 大会员、YouTube 18+）需要 Cookie：

1. 在浏览器安装 **"Get cookies.txt locally"** 扩展。
2. 登录视频网站。
3. 导出 Cookie 为 `cookies.txt` 文件。
4. 在 VDD 设置中选择该文件。

如果视频解析失败，可能是访问受限或登录态缺失，此时可以尝试设置 Cookie 来解决。

### 3.4 并发数

- 建议设置为 **3** 或 **5**。设置过高可能会被目标网站限速或封禁 IP。

## ❓ 5. 常见问题 (FAQ)

- **Q: 下载失败怎么办？**

请检查网络连接。如果是海外平台（如 YouTube），请在设置中配置代理（例如 <http://127.0.0.1:7890>, 具体需要依赖你的代理软件）。

- **Q: 只有声音没有画面？**

请确保下载的是 Video 或 Video+Audio 格式。

- Q: **输入链接，解析失败**

请检查链接是否正确, 如果是海外平台（如 YouTube），请在设置中配置代理（例如 <http://127.0.0.1:7890>, 具体需要依赖你的代理软件）。另外，可以设置 Cookie，以访问一些需要登录才能访问的视频。

- Q: **解析为什么这么久？**

VDD 需要请求网站并解析出视频元数据信息，这个过程通常需要几秒到几分钟不等。通常，国内网站速度会比较快，通常几秒就可以解析出结果。如果是国外网站可能耗时比较久，请耐心等待。如果解析仍然失败或者超时，可以尝试设置cookie再重试。

- **Q: 为什么有的平台下载速度很慢？**

有的网站下载速度很快，但是有的却很慢，这通常与网站访问速度和限流有关，可以考虑登录视频源网站并设置cookie。

- **Q: 设置保存时提示 "XML syntax error"？**

这是界面库的无害日志警告，只要设置窗口关闭后更改生效了，可以忽略它。

- **Q: 软件无法打开 (macOS)？**

显示“文件已损坏”或“无法验证开发者”时，请在 `系统设置 -> 隐私与安全性` 中允许任何来源，或在终端执行 `sudo xattr -r -d com.apple.quarantine /Applications/VDD.app`。

- Q: Windows安装提示有病毒?

这属于杀毒软件的误报。郑重承诺：本软件代码全部开源，软件本身未包含任何恶意代码, 请放心使用。

- Q: 如何设置cookie?

系统设置中已经支持从不同的浏览器中解析 Cookie 数据，建议优先使用这种方式。

如果您想要手动设置Cookie，这里以 Chrome 浏览器为例，其他浏览器请自行搜索对应插件。

1. 应用商店安装 get cookies.txt locally 插件

   ![](../assets/cookie/访问chrome应用商店.png)
   在打开的页面中，搜索 get cookies.txt locally。
   ![](../assets/cookie/搜索插件.png)
   然后点击 添加到 Chrome，等待安装完成。
   ![](../assets/cookie/安装插件.png)

2. 使用插件导出 Netscape 格式的 cookie

   安装完成后，访问你想要的视频网站时，点击插件图标，即可看到当前的 cookie 信息，比如我访问的 b 站:
   ![](../assets/cookie/插件窗口.png)
   接下来，需要导出，Export Format 选择 Netscape，然后点击 Export 即可，通常，导出的文件放在下载目录中。

3. VDD 中设置 cookie

打开 VDD，点击首页解析url输入框的右上侧的饼干图标，然后点击 浏览，选择你刚才导出的 cookie 文件, 后缀为 xxx_cookies.txt，最后千万别忘了点击 保存按钮。
